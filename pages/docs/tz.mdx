# About the Project

# Технічне завдання (ТЗ)

## 1. Загальні положення
Мета – створення API платформи електронної комерції для управління замовленнями та оплатами.

## 2. Призначення системи
Система призначена для інтеграції з клієнтськими інтерфейсами для обробки замовлень і платежів у форматі REST.

## 3. Функціональні вимоги
- CRUD-операції для `Order` та `Payment`
- Валідація даних
- Вивід помилок із відповідними статус-кодами

## 4. Нефункціональні вимоги
- Надійність API
- Документація доступна онлайн
- Відповідність OpenAPI 3.0

## 5. Інтерфейси
- Вхідні: HTTP-запити (JSON)
- Вихідні: HTTP-відповіді з кодами та JSON-даними

## 6. Операційне середовище
- Сервер: GitHub Pages
- Клієнт: будь-який HTTP-клієнт

## 7. Документація
- YAML OpenAPI 3.0
- Zudoku генератор документації

## 8. Розгортання
- GitHub Pages, інтегрований з main-гілкою
